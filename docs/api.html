CLI & API Reference

Complete documentation for developers and AI agents

CLI Usage

python research_agent.py [query] [--flags]

Options

FlagTypeDefaultDescription
querystring—Research query (positional, optional)
--evalflagfalseRun evaluation suite
--eval-runsint2Runs per eval task
--gepaflagfalseRun GEPA self-improvement loop
--gepa-generationsint3Number of GEPA generations
--gepa-poolint4Candidates per generation
--dogfoodflagfalseFull pipeline: eval + GEPA + self-patch
--searx-instancestring—SearXNG instance URL (e.g. http://localhost:8888)
--self-updateflagfalseUpdate research-agent.py from GitHub
--statestringresearch_state.jsonPath to state file
--forceflagfalseRe-run even if completed
--clearflagfalseClear saved state and exit
--reportflagfalsePrint last report and exit

Examples

# Basic research
python research_agent.py "What are the latest developments in RAG?"

# Run eval suite (3 tasks × 4 checks each)
python research_agent.py --eval --eval-runs 1

# Self-improvement loop
python research_agent.py --gepa --gepa-generations 5 --gepa-pool 6

# Full dogfood pipeline
python research_agent.py --dogfood

# Use custom SearXNG instance
python research_agent.py --searx-instance http://searxng.local:8888 "latest AI papers"

# Update to latest version
python research_agent.py --self-update

# Clear state and re-run
python research_agent.py --clear

# View last report
python research_agent.py --report

Programmatic API

Import and use research-agent as a Python module:

ResearchAgent

from research_agent import ResearchAgent
import asyncio

agent = ResearchAgent("What is the impact of AI on healthcare?")
state = asyncio.run(agent.run())
print(state.report[:500])
print(f"Status: {state.status}")
print(f"Errors: {len(state.error_log)}")

EvaluationRunner

from research_agent import EvaluationRunner

runner = EvaluationRunner(n_runs=2)
results = asyncio.run(runner.run_all())
for task in results.get("tasks", []):
    title = task["title"]
    errors = sum(r.get("errors", 0) for r in task.get("runs", []))
    print(f"{title}: {'PASS' if errors == 0 else 'FAIL'}")

GEPALoop

from research_agent import GEPALoop, EvaluationRunner

gepa = GEPALoop(pool_size=4, max_generations=3)
eval_runner = EvaluationRunner(n_runs=1)
results = asyncio.run(gepa.run_self_improvement(eval_runner))
print(gepa.format_report(results))

SessionManager

from research_agent import SessionManager

sm = SessionManager("my_state.json")
state = sm.load()
if state and state.report:
    print(state.report)
sm.clear()  # Delete saved state

Eval Suite

The evaluation suite tests research quality across 3 tasks:

IDTaskChecks
eval-001Ambiguous Health Claim — "What are the proven benefits of intermittent fasting?"Executive summary, findings with sources, bib with citations, contradictions section
eval-002Emerging Technology Impact — "How is AI being used in climate change research?"Executive summary, findings with sources, bib with citations, contradictions section
eval-003Historical Revisionism — "What caused the fall of the Roman Empire? Compare economic, military, and political factors."Executive summary, findings with sources, bib with citations, contradictions section

Each check is a keyword-based heuristic scan of the generated report. A report must contain specific expected sections and patterns to pass. All checks are deterministic and fast (~ms per check).

GEPA Self-Improvement

The GEPA loop evolves prompt strategies to maximize report quality:

PromptCandidate

@dataclass
class PromptCandidate:
    id: str  # e.g. "gen0-0"
    prompt_template: dict[str, str]  # One prompt per module
    accuracy: float = 0.0
    avg_speed_ms: float = 0.0
    generation: int = 0

Lesson

Reflections produce structured lessons that drive targeted mutations:

@dataclass
class Lesson:
    module: str          # Which module needs improvement
    check_name: str      # Which check failed
    suggestion: str      # What to change in the prompt

Mutation Strategy

  • If a lesson targets a specific module, only that module's prompt is mutated
  • If no lessons, a random module is mutated to explore
  • If all modules have issues, all prompts are regenerated
  • Winner is selected by accuracy (Pareto-optimal: max accuracy, then max speed)

Grading System

Two-tier grading pipeline:

GraderMethodDependencies
HeuristicGraderKeyword-based report quality analysisNone (built-in)
OllamaGraderLocal LLM via Ollama APIOllama installed + model pulled
ModelGraderRouter: tries Ollama, falls back to heuristicAuto-detected

The heuristic grader checks: executive summary length, finding count with citations, source diversity, contradiction analysis depth, and conclusion quality. Returns 0.0-1.0 score + structured feedback.

Self-Patching

After a GEPA run, the best candidate's prompts are written to _self_patch.py:

# Apply learned improvements
python _self_patch.py
# Patched to v2.X.X with N prompt updates

The patch script modifies research_agent.py in-place, updating SCRIPT_VERSION and modifying the BASE_PROMPTS dictionary. The original file is backed up by git (the patch only makes surgical replacements).
